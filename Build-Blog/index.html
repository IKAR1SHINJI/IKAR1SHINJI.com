用Hexo+Github搭建个人博客

发表于 2017-11-01 | 更新于: 2018-06-16 | 阅读次数:

Hexo是一款基于 Node.js 的静态博客框架，可以方便地生成静态网页并免费托管在GitHub 或Coding 上，实现不花一分钱搭建自己的独立博客。

搭建步骤：

  • 创建个人仓库
  • 配置开发环境
  • 部署网站
  • 编写博文

参考资料：
Hexo开发文档
GitHub+Hexo 搭建个人网站详细教程

创建个人仓库

注册并登录GitHub,点击New repository创建新仓库，仓库名为：用户名.github.io

配置开发环境

需要安装的软件有 Git，Node.js 和 Hexo。

安装Git

Git 是开源的分布式版本控制系统，在本地搭建好网站后需要使用 Git 同步到 GitHub 上。下载Git 并安装后，在命令行输入git测试是否安装成功。

安装成功后，鼠标在资源管理器右击打开Git Bash。

设置user.name和user.email配置信息,将 Git 与 GitHub 帐号绑定：

生成ssh密钥文件：

然后直接三个回车即可，默认不需要设置密码，在C:\Users\用户名\.ssh找到id_rsa.pub密钥文件，用记事本打开并复制全部内容。

打开SSH设置页面 ，new SSH Key。

Title 任意填写即可，将刚刚复制的id_rsa.pub中的内容粘贴进Key，最后点击 Add SSH key。

在 Git Bash 中检测 GitHub 公钥设置是否成功，输入ssh git@github.com

如上则说明成功。

这里稍作解释，GitHub要求每次推送代码都是合法用户，这里通过ssh(安全外壳协议)验证你电脑中的私钥与GitHub上面的公钥是否配对来确认你合法用户的身份，避免了每次输入密码的麻烦。

如果想要了解Git的更多内容，参看Git教程 。

编写博文

之前我们只是发布了一篇没有正文的博文，那么如何编写博文内容呢？

首先我们要找到博文的位置，打开~\blog\source\_post\找到创建的HelloWorld.md文件，”.md”后缀说明这种文件是Markdown格式。用记事本打开，看看里面的内容：

Front-matter 部分参数如下：

参数 描述 默认值
layout 布局
title 标题
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
tags 标签（不适用于分页）
categories 分类（不适用于分页）
comments 开启文章的评论功能 true

认识Markdown

Markdown 是一种轻量级标记语言，允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档”。我们正是通过编写 Markdown 格式的文档来实现博文的编写与排版。

工欲善其事，必先利其器。虽然 Markdown 是纯文本格式，使用记事本便可以打开，但为了高效编写与预览博文，推荐使用MarkdownPad2 。

Markdown语法

介绍一些常用的Markdown语法：

功能 语法 快捷键

斜体 *斜体* Ctrl+I
引用 > 引用 Ctrl+Q
行内代码 `行内代码` Ctrl+K
代码段 每一行代码前加上制表符(Tab) Ctrl+K
